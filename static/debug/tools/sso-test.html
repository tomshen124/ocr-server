📝 功能说明

回调模式（useCallback: true）：
- 构造包含回调参数的SSO URL
- 第三方登录后会尝试回调到我们的系统
- 适用于生产环境（需要公网域名）
- 本地开发时会因为127.0.0.1地址导致404错误
直跳模式（useCallback: false）：
- 只包含appId参数的简单SSO URL
- 用户登录后需要手动返回系统
- 适用于本地开发和测试
- 避免了回调地址的404问题

🔧 配置建议

本地开发：建议使用 useCallback: false 或启用Mock登录
生产环境：使用 useCallback: true 并配置正确的公网回调地址
